# Plán vylepšení MD Helper — Top 5 priorit

## Cíl
Implementace 5 vybraných funkcí s nejvyšším poměrem užitku / náročnosti pro jednostránkovou HTML aplikaci MD Helper.

## Kritéria výběru
- Okamžitý UX prospěch pro běžného uživatele
- Nízká implementační náročnost (vše v rámci jednoho HTML souboru)
- Žádné externí závislosti navíc
- Kompatibilita se stávajícím dark-only designem

---

## Top 5 — seřazeno podle priority implementace

### 1. Toast notifikace (Fáze 1.1)
**Proč první:** Nahradí všechny `alert()`, což je okamžitě viditelné zlepšení.
- Nahradit `alert()` při chybě drag & drop a fallbacku kopírování.
- Přidat potvrzovací toast i pro úspěšné akce (kopírování, stáhnutí, export).
- Implementace: vlastní CSS/JS toast kontejner (fixed position, auto-dismiss po 3s).
- **Soubor:** `md-helper.html` (JS + CSS)

### 2. Split view přepínač — Editor / Editor+Preview / Preview (Fáze 1.3)
**Proč druhé:** Zásadně zlepší focus při psaní i čtení.
- Tlačítko v toolbaru pro přepínání mezi 3 režimy:
  - **Editor + Preview** (současný stav)
  - **Pouze Editor** (skryje preview, rozšíří editor na 100% šířky)
  - **Pouze Preview** (skryje editor — užitečné pro čtení výsledku)
- Při pouze-editor/preview režimu přepnout grid na 1 sloupec.
- Uložit preferenci do `localStorage`.

### 3. Find & Replace (Fáze 2.1)
**Proč třetí:** Nezbytný editorový nástroj, který chybí.
- Vlastní `Ctrl+F` / `Ctrl+H` dialog (ne nativní prohlížečový, aby fungoval v textarea).
- UI: vyhledávací pole + nahradit pole + tlačítka Najít další / Nahradit / Nahradit vše.
- Implementace přes `editor.value` a `setSelectionRange`.
- Zvýraznění všech výskytů + počet nalezených.

### 4. Drag & drop obrázků s base64 embed (Fáze 3.2)
**Proč čtvrté:** Rozšíří MD Helper z čistě textového editoru na rich content editor.
- Rozšířit stávající D&D logiku i pro obrázky (`.png`, `.jpg`, `.jpeg`, `.gif`, `.webp`, `.svg`).
- Přečíst soubor jako `FileReader.readAsDataURL()`.
- Vložit do editoru jako `![název](data:image/...)` na aktuální pozici kurzoru.
- Varovný toast při obrázku > 1MB (base64 může být obrovské).
- Zároveň opravit drag & drop `.md` — nyní se kontroluje jen `.endsWith('.md')`, což nefunguje pro `.MD`.

### 5. Resizable panely — přetahování divideru (Fáze 2.2)
**Proč páté:** Umožní uživateli přizpůsobit si poměr editor / náhled podle potřeby.
- Mezi editorem a preview přidat draggable vertikální divider (3px široký, indigo barva).
- Upravit šířku sloupců pomocí CSS custom properties (`--editor-width`).
- Uložit preferenci do `localStorage`.
- Respektovat responzivní breakpoint (<1024px se divider skryje).

## Co záměrně vynecháváme (prozatím)

| Funkce | Důvod vynechání |
|--------|-----------------|
| Timestamp posledních úprav | Menší přínos, word count stačí |
| Markdown validace | Příliš komplexní bez plnohodnotného parseru |
| Export do PDF | `window.print()` je dostatečný workaround |
| Slovník zkratek | Užitečné, ale pro uživatele neintuitivní bez UI správy |
| Light/dark theme | Původní zadání je dark-only |
| Minimap | Náročné na implementaci, malý přínos pro krátké MD soubory |
| PWA manifest + Service Worker | Vyžaduje další soubory mimo jedno HTML |
| Web Worker pro parsing | Předčasná optimalizace, současné soubory nejsou tak velké |

## Implementační poznámky

- Všechny změny zůstanou v jednom souboru `md-helper.html` (žádné nové soubory).
- Použijeme `localStorage` pro perzistenci preferencí (split view, šířka panelů).
- Žádné nové externí knihovny — vše čisté JS/CSS.
- Zachováme stávající dark theme, žádný refaktor barev.
